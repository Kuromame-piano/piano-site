# NEWS機能の静的化完了ガイド

microCMSを使わない、静的なNEWS機能に変更しました。

## 変更内容

### 削除したファイル
- `src/lib/microcms.ts` - microCMSクライアント
- `src/pages/news/[blogId].astro` - 動的記事ページ
- `package.json`から`microcms-js-sdk`を削除
- `.env`のmicroCMS設定をコメントアウト

### 変更したファイル
- `src/pages/news/index.astro` - 静的な記事一覧ページに変更
- `src/pages/index.astro` - microCMS呼び出しを削除

### 新規作成したファイル
- `src/pages/news/about.astro` - 最初の記事（教室概要説明）

## 新しい記事の追加方法

記事を追加したい時は、以下の手順で行います：

### 1. 記事コンテンツを準備

記事の内容（タイトル、本文、公開日）をテキストで準備してください。

### 2. AIに依頼

準備した内容を私（AI）に送ってください。以下の形式で送っていただくと作業がスムーズです：

```
新しいニュース記事を作成してください。

タイトル: （記事タイトル）
公開日: 2026-06-25
スラッグ: new-piano-lesson（URLになる部分）

本文:
（記事の内容をここに記載）
```

### 3. ファイル作成と一覧更新

AIが以下の作業を行います：
- `src/pages/news/スラッグ.astro` を作成
- `src/pages/news/index.astro` の記事一覧を更新

### 4. 確認とデプロイ

- ローカルで `npm run build` を実行して確認
- 問題なければデプロイ

## 記事一覧の管理

`src/pages/news/index.astro` の `newsItems` 配列に記事情報が入っています：

```javascript
const newsItems = [
    {
        slug: "about",
        title: "大人向け出張ピアノレッスンのご案内",
        publishedAt: "2026-06-18",
    },
    // 新しい記事を追加する場合はここに追加
];
```

新しい記事を追加する時は、この配列の**先頭**に追加します（新しい記事が上に表示されるため）。

## メリット

この静的方式のメリット：
- microCMSの月額料金が不要
- すべてGitで管理できる（変更履歴が残る）
- ビルド時に完全静的化されるのでパフォーマンスが良い
- セキュリティリスクが減る
- シンプルで管理しやすい

## 注意事項

- 記事の追加・編集にはコードの変更とビルドが必要です
- 頻繁に記事を更新する場合は、CMSの方が便利な場合もあります
- 現在の運用スタイル（記事を頻繁に投稿しない）には最適です
